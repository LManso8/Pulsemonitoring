# Graphical User Interface & Visualization Engine

## Overview
The Visualization module constitutes the presentation layer of the GeoNode architecture. It operates on a hybrid rendering model: it utilizes `Tkinter` to maintain a low-latency, event-driven interactive dashboard, while asynchronously calling `Matplotlib` to render complex scientific arrays generated by the DSP and Interpretation engines. 

To ensure optimal performance and avoid "Disk I/O bottlenecks", all visual components are rendered **in-memory** for immediate on-screen analysis, with persistent file exports executed exclusively upon explicit operator command.

## Sub-Module Architecture

The rendering logic is strictly decoupled into the two fundamental domains of signal analysis:

### 1. Time-Domain Rendering (`amplitude_time_plot.py`)
* **Purpose:** Visualizes the kinematic progression of the structural event (Seismogram).
* **Rendered Elements:** * Plots the integrated Velocity (mm/s) arrays for the $V_x, V_y, V_z$ axes over relative time ($\Delta t$).
  * Injects dynamic horizontal threshold lines (Alert Limits) calculated by the normative Interpretation modules directly onto the kinematic curves to provide immediate visual compliance verification.

### 2. Frequency-Domain Rendering (`frequency_plot.py`)
* **Purpose:** Visualizes the spectral energy distribution to identify structural resonance.
* **Rendered Elements:** * Plots the output arrays of the Fast Fourier Transform (FFT).
  * Automatically isolates and annotates the Dominant Frequency peak (Hz) against the amplitude spectrum.

### 3. Interactive Dashboard (`dashboard_gui.py`)
* **Purpose:** The centralized control center bridging human operation and backend telemetry.
* **Features:** Live event feed polling, normative mode toggling (Environmental vs. Seismic), and asynchronous trigger controls for report generation.

## Export Capabilities
While primary rendering is strictly on-demand and in-memory, the module supports standard physical exports for compliance auditing and peer review.
* **Vector Formats:** PDF (Preferred for high-fidelity technical reports).
* **Raster Formats:** PNG, JPG (Supported for quick distribution).
